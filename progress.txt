## 2026-02-10 - Bug Fix: 视频/图片生成卡在"生成中"状态

### 问题描述:
用户报告视频一直显示"生成中"状态，无法完成。刷新页面后状态仍然显示"生成中"。

### 根本原因:
视频/图片生成的轮询机制运行在客户端。当用户导航离开页面、刷新页面或关闭浏览器时，轮询停止。即使 Volc Engine 已经完成生成任务，由于没有轮询检查状态，数据库中的 video_status/image_status 不会更新。

### What was done:
1. **components/scene/SceneVideoList.tsx**:
   - 添加 useEffect 钩子，在组件加载时检查是否有 video_status === "processing" 的分镜
   - 如果有，自动恢复这些视频的轮询

2. **components/scene/SceneImageList.tsx**:
   - 添加 useEffect 钩子，在组件加载时检查是否有 image_status === "processing" 的分镜
   - 如果有，自动恢复这些图片的轮询

### Testing:
- ✅ npm run lint 通过
- ✅ 手动调用视频任务状态 API 确认视频已生成并下载
- ✅ 刷新页面后视频状态正确显示为"已完成"

### Notes:
- 此修复确保即使用户离开页面，下次回来时也会自动恢复轮询
- 对于已经卡住的任务，页面加载时会自动检查并更新状态
- 视频任务状态 API (`/api/generate/video/task/:taskId`) 会在检查时自动下载并保存视频

   - 确保火山引擎可以访问私有 bucket 中的图片

2. **app/api/generate/videos/route.ts**:
   - 同样使用 `getSignedUrl()` 替代公共 URL

3. **components/scene/SceneVideoList.tsx**:
   - 修改 `handleGenerateVideo` 捕获 API 返回的 taskId 和 videoId
   - 修改 `pollForVideoCompletion` 调用视频任务状态 API:
     - GET /api/generate/video/task/{taskId}?sceneId=...&projectId=...&videoId=...
   - 修改 `handleGenerateAll` 使用批量 API 返回的任务 ID 进行轮询

### Testing:
- ✅ npm run lint 通过
- ✅ npm run build 通过
- ✅ 浏览器测试确认：
  - 视频生成 API 调用成功

### Notes:

## 2026-02-10 - Bug Fix: Storage 签名 URL 修复

### 问题描述:
用户报告生成的图片在网页上显示失败，网络请求返回 400 错误。

### 根本原因:
- Supabase Storage bucket `project-media` 被设置为私有 (`public: false`)
- 代码使用 `getPublicUrl()` 获取图片 URL，这只适用于公开 bucket
- 私有 bucket 需要使用签名 URL (`createSignedUrl()`)

### What was done:
1. **lib/db/media.ts** - 添加签名 URL 函数:
   - `getSignedUrl()` - 为单个文件生成签名 URL
   - `getSignedUrls()` - 批量生成签名 URL
   - 更新 `uploadFile()` 返回签名 URL 而非公开 URL

2. **app/api/storage/signed-urls/route.ts** - 创建新 API 端点:
   - POST /api/storage/signed-urls - 批量获取签名 URL
   - 验证用户身份和路径所有权

3. **hooks/useSignedUrls.ts** - 创建 React Hook:
   - `useSignedUrls()` - 批量获取签名 URL
   - `useSignedUrl()` - 获取单个签名 URL
   - 自动刷新即将过期的 URL

4. **更新组件使用签名 URL**:
   - SceneImageCard.tsx - 添加 `signedUrl` prop
   - SceneImageList.tsx - 使用 `useSignedUrls` hook
   - SceneVideoCard.tsx - 添加 `signedImageUrl` 和 `signedVideoUrl` props
   - SceneVideoList.tsx - 使用 `useSignedUrls` hook
   - CompletedProjectView.tsx - 使用 `useSignedUrls` hook
   - lib/db/projects-list.ts - 服务端获取签名 URL 用于项目预览图

### Testing:
- ✅ TypeScript 类型检查通过
- ✅ npm run lint 通过
- ✅ 浏览器测试通过（使用 MCP Playwright）:
  - 项目列表页预览图正确显示
  - 项目详情页分镜图片正确显示
  - 签名 URL API 返回 200 OK
  - 图片加载成功（签名 URL 格式：`/storage/v1/object/sign/...?token=...`）

### Notes:
- 签名 URL 默认有效期 1 小时
- `useSignedUrls` hook 会在 URL 过期前自动刷新（80% 过期时间时）
- 初始渲染时会短暂显示失败（使用数据库存储的旧 URL），但会立即被签名 URL 替换
- 服务端渲染的组件（如项目列表）直接获取签名 URL，无需客户端 hook
