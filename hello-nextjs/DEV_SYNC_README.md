# 浏览器到源码同步系统 (Browser-to-Source Sync)

## 功能说明

本系统实现了在浏览器中修改页面元素后，可以自动将这些修改同步到源代码文件的功能。

## 架构

```
┌─────────────────────────────────────────────────────────────┐
│                     浏览器开发者工具                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  MutationObserver 监听 DOM 变化                      │   │
│  │  (属性修改、文本修改、子节点变化)                       │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  同步面板 (Ctrl+Shift+S)                             │   │
│  │  标记模式 (Ctrl+Shift+M) - 点击元素标记到源文件        │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                 │
│              POST /api/dev-sync                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                    Next.js 开发服务器                         │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  API Route: /api/dev-sync                            │   │
│  │  - 接收浏览器修改请求                                  │   │
│  │  - 解析源代码文件                                      │   │
│  │  - 应用修改并保存                                      │   │
│  └─────────────────────────────────────────────────────┘   │
│                            ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Source Analyzer (src/lib/dev-sync/)                │   │
│  │  - AST 解析                                          │   │
│  │  - React/JS/CSS 修改                                │   │
│  │  - 代码生成                                          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                       源代码文件                              │
│  src/app/page.tsx  ← 修改保存到这里                          │
│  src/components/*.tsx                                       │
│  src/**/*.css                                               │
└─────────────────────────────────────────────────────────────┘
```

## 使用方法

### 1. 启动开发服务器

```bash
npm run dev
```

这会自动：
- 复制浏览器开发工具脚本到 `public/scripts/`
- 启动 Next.js 开发服务器
- 注入 DevSyncProvider 到所有页面

### 2. 打开浏览器开发者工具

访问 http://localhost:3000，打开浏览器的开发者工具（F12 或右键 → 检查）。

### 3. 打开同步面板

- 按 `Ctrl+Shift+S` 或在控制台查看日志

### 4. 标记元素到源文件

有两种方式标记元素：

#### 方式一：标记模式
1. 按 `Ctrl+Shift+M` 进入标记模式
2. 在页面上点击要标记的元素
3. 输入对应的源文件路径（如 `src/app/page.tsx`）
4. 元素会被标记 `data-dev-id` 属性

#### 方式二：手动添加标记
在源代码中添加 `data-dev-path` 属性：

```tsx
<div data-dev-id="header-title" data-dev-path="src/app/page.tsx">
  我的标题
</div>
```

### 5. 修改元素

在浏览器开发者工具中直接编辑：
- 修改文本内容
- 修改 class 属性
- 修改 placeholder、value 等属性
- 添加/删除子元素

### 6. 同步到源码

修改会自动同步（500ms 防抖），也可以：
- 点击同步面板的"立即同步"按钮
- 页面会自动刷新查看新代码

## 支持的文件类型

| 文件类型 | 支持的修改 | 示例 |
|---------|-----------|------|
| `.tsx` / `.jsx` | 属性、文本、样式 | `<div className="...">` |
| `.ts` / `.js` | 字符串值 | `const title = "..."` |
| `.css` / `.scss` | CSS 规则 | `.class { color: ... }` |
| `.json` | JSON 字段 | `{ "key": "value" }` |

## 配置文件

### 浏览器脚本配置

在 `scripts/browser-dev-tools.js` 中修改：

```javascript
const DEV_SYNC_CONFIG = {
  enabled: true,              // 是否启用
  apiEndpoint: '/api/dev-sync', // API 端点
  debounceMs: 500,            // 防抖延迟
  consoleOutput: true        // 控制台输出
};
```

### 标记数据

标记信息保存在 `.dev-markers.json`：

```json
{
  "element-1": {
    "id": "element-1",
    "path": "src/app/page.tsx",
    "selector": "[data-dev-id='element-1']",
    "attributes": {},
    "lastModified": 1709123456789
  }
}
```

## 工作流程示例

### 示例 1：修改标题文本

1. 在浏览器中点击标题元素，右键 → 检查
2. 在 Elements 面板中找到对应的元素
3. 添加属性：`data-dev-path="src/app/page.tsx"`
4. 直接双击文本进行编辑
5. 保存后源码会自动更新

### 示例 2：修改样式类名

1. 标记元素并指定文件路径
2. 在 Styles 面板修改 class
3. 修改会自动同步到源文件

### 示例 3：修改表单占位符

1. 标记 input 元素
2. 在开发者工具中修改 placeholder 属性
3. 源码中的 `placeholder` 属性会自动更新

## 限制和注意事项

⚠️ **重要限制**：

1. **仅开发环境**：生产环境会自动禁用
2. **简单修改优先**：复杂的结构变化可能需要手动调整
3. **需要标记**：
   - 使用标记模式
   - 或手动添加 `data-dev-path` 属性
4. **防抖机制**：避免频繁请求（默认 500ms）
5. **备份建议**：修改前建议备份文件或使用 Git

## 故障排除

### 修改没有生效

1. 检查浏览器控制台是否有错误
2. 确认元素有 `data-dev-id` 或 `data-dev-path` 属性
3. 检查文件路径是否正确
4. 查看终端中的服务器日志

### API 返回错误

1. 检查文件是否存在
2. 确认文件路径格式正确（相对于 `src/` 目录）
3. 查看 Next.js 控制台错误信息

### 热重载不工作

1. 确保 Next.js 开发服务器正在运行
2. 检查 `.next` 缓存，必要时删除并重启

## 技术细节

### MutationObserver 监听的事件

- `attributes`：属性变化（class、style、value 等）
- `childList`：子节点添加/删除
- `subtree`：监听所有后代元素

### API 端点

**POST /api/dev-sync**
```json
{
  "modifications": [
    {
      "type": "attributes",
      "devId": "element-1",
      "devPath": "src/app/page.tsx",
      "element": {
        "tagName": "DIV",
        "attributes": {
          "className": "new-class"
        }
      }
    }
  ],
  "url": "http://localhost:3000/",
  "timestamp": 1709123456789
}
```

### 源码解析

使用字符串匹配和正则表达式来定位和修改代码：
- React JSX 属性修改
- CSS 类名替换
- 文本内容替换
